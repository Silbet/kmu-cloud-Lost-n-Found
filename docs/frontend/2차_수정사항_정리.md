# 2차 수정사항

## 1. 운영 관리 대시보드 - 상태별 박스 클릭 시 필터링 적용

**경로:** 운영자 로그인 → 운영 관리 대시보드

**현재 동작:**
- '분실 신고 상태'와 '습득물 상태' 섹션에 상태별로 세분화된 하위 박스가 여러 개 존재
- 어떤 박스를 클릭해도 각각 '전체 분실 신고', '전체 습득물' 페이지로 이동함

**문제:**
박스는 상태별로 나뉘어 있는데, 클릭 시에는 전체 목록만 보여줌

**수정 요청:**
각 박스 클릭 시 해당 상태로 **자동 필터링된 페이지**로 이동하도록 수정
- 예: '처리 중' 박스 클릭 → 처리 중 상태로만 필터링된 목록 표시
- 예: '보관 중' 박스 클릭 → 보관 중 상태로만 필터링된 목록 표시

---

## 2. 운영자 검색 메뉴 - 기본값을 전체 물품 보기로

**경로:** 운영자 로그인 → 검색 메뉴

**현재 동작:**
검색 전에는 물품이 하나도 표시되지 않음

**수정 요청:**
검색 전 기본 상태로 **전체 물품 목록이 표시**되도록 변경

---

## 3. 검색 결과 클릭 시 상세 정보 + 신고자/습득자 정보 표시

**경로:** 운영자 로그인 → 검색 메뉴 → 물품 클릭

**수정 요청:**
물품을 클릭했을 때 다음 정보가 함께 표시되도록 수정
- 물품 관련 정보 (기존)
- **신고자 정보** (분실물의 경우)
- **습득자 정보** (습득물의 경우)

---

## 4. 분실 신고 작성 - 분실 장소를 선택 사항으로 변경

**경로:** 분실 신고 작성 페이지

**수정 요청:**
'분실 장소' 입력 필드를 **필수 → 선택 사항**으로 변경
- 폼 유효성 검사(validation) 로직에서 해당 필드의 required 속성 제거
- 빈 값으로 제출해도 정상 등록되도록 처리

---

## 5. 일반 사용자 분실물 검색 - 개인정보 마스킹 처리

**경로:** 일반 사용자 로그인 → 분실물 검색

**문제:**
다른 사람의 개인정보가 그대로 노출되고 있음

**수정 요청:**
다른 사용자의 개인정보를 **마스킹 처리**하여 표시
- 이름: `홍길동` → `홍*동`
- 연락처: `010-1234-5678` → `010-****-5678`
- 이메일: `user@example.com` → `u***@example.com`

> 운영자 화면에서는 마스킹하지 않고 원본 그대로 표시 (현행 유지)

---

## 6. 연락처 입력 시 자동 하이픈 처리

**적용 범위:** 연락처를 입력하는 모든 필드 (회원가입, 신고 작성, 정보 수정 등 전체)

**수정 요청:**
사용자가 숫자만 입력하면 **자동으로 하이픈이 삽입**되도록 처리
- 최종 형식: `010-1234-5678`
- 동작 예시:
  - 입력: `01012345678`
  - 표시: `010-1234-5678`
- 입력 중에도 실시간으로 하이픈이 자동 삽입되도록 처리 (onChange 기준)

---

## 7. 테스트 계정 로그인 오류 수정

**문제 상황:**
다음 테스트 계정으로 로그인이 되지 않음
- **일반 사용자** 테스트 계정
- **승인 대기 관리자** 테스트 계정

**수정 요청:**
두 계정 모두 정상 로그인되도록 원인 파악 및 수정

**점검 사항:**
- 시드 데이터(seed data)에 해당 계정이 정상적으로 등록되어 있는지
- 비밀번호 해싱 방식이 로그인 검증 로직과 일치하는지
- 승인 대기 관리자의 경우, 로그인 시 별도 분기 처리(승인 대기 안내 메시지 등)가 정상 작동하는지
- 계정 활성화 상태(is_active 등) 플래그 확인
