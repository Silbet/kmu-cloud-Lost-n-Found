# 분실물/습득물 관리 시스템 수정사항

> Claude Code 전달용 작업 명세서

---

## 📌 작업 우선순위 요약

1. **🔴 긴급 (버그/오류)**: 검색 버튼 텍스트 잘림, 로그인 403 오류
2. **🟡 구조 변경 (Breaking Changes)**: 회원가입/계정 체계 개편, 매칭 완료 상태 구조 변경
3. **🟢 기능 추가/UX 개선**: 그 외 항목

---

## 1. 회원가입 / 계정 체계 개편 ⚠️ Breaking Change

### 1-1. 계정 유형 단순화
- **현재**: 분실자 / 습득자 / 분실+습득 겸직 등으로 구분
- **변경**: 다음 2가지 역할로 통합
  - `일반 사용자` (기존 "분실+습득 겸직" 계정의 화면 구성을 기본값으로 사용)
  - `보관소 관리자`
- 회원가입 시 위 2개 중 **1개만 선택** 가능
- **삭제 대상**: 분실자 단독, 습득자 단독 계정 및 관련 화면 구성

### 1-2. 보관소 관리자 가입 승인 플로우 신설
- 보관소 관리자로 회원가입 신청 시 즉시 접근 권한 부여하지 않음
- **운영자 승인** 후에만 접근 권한 활성화
- 운영자에게 **승인 요청 알림** 발송
- 운영자 대시보드에 **"보관소 관리자 회원가입 승인 페이지"** 신규 생성

### 1-3. 운영자 계정 생성 방식
- 신규 운영자 계정은 **회원가입 화면이 아닌 초기 운영자 대시보드에서 생성**
- 초기 운영자 대시보드의 적절한 위치에 **"운영자 계정 생성" 버튼** 추가
- 초기 운영자가 회원가입 정보(이메일/비밀번호)를 직접 입력하여 계정 생성 및 권한 부여
- 생성된 이메일/비밀번호는 초기 운영자가 신규 운영자에게 직접 전달하는 방식

### 1-4. 이메일 필드 명칭 변경
- 회원가입 및 로그인 화면의 `이메일` → **`학교 이메일`** 로 명칭 교체

### 1-5. 마이페이지 신설
- 모든 회원가입 사용자가 로그인 후 접근 가능한 **상단 메뉴 "마이페이지"** 신규 추가
- 마이페이지 내 **비밀번호 변경** 기능 포함

## 2. 로그인 / 초기 화면

### 2-1. 로그인 후 403 오류 해결 🔴
- 로그인 직후 초기 화면에서 **"403 이 페이지에 접근할 권한이 없습니다"** 오류 표시되는 문제 수정
- 다른 메뉴 버튼 클릭 시에는 정상 작동하므로 **초기 진입 라우팅/권한 체크 로직** 점검 필요

### 2-2. 역할별 홈 화면 라우팅
- **운영자 로그인 시**: 운영자 관리 대시보드를 메인 홈/초기 화면으로 표시
- **보관소 관리자 로그인 시**: 보관소 관리자 대시보드를 메인 홈/초기 화면으로 표시

---

## 3. 검색 메뉴 UX 개선

### 3-1. 메뉴 명칭 보조 문구 추가
- **분실물 검색 화면**: `"이걸 잃어버렸어요"` 문구 추가
- **습득물 검색 화면**: `"이걸 찾았어요"` 문구 추가
- 각 화면 진입 시 해당 화면에서 어떤 동작을 할 수 있는지 **짧은 설명 문구** 작게 표시

### 3-2. 검색 버튼 텍스트 잘림 수정 🔴
- 필터 선택 후 누르는 **`검색` 버튼의 한글이 잘려서 보이지 않음** → 수정

### 3-3. 초기화 버튼 추가
- `검색` 버튼 **옆**에 **`초기화`** 버튼 신규 생성 (필터 초기화 용도)

### 3-4. 물품 상태 표시 설명 추가
- 검색 결과 물품들에 표시되는 상태(예: 매칭후보있음, 보관중 등)가 어떤 의미인지
- **각 상태에 대한 짧은 설명**을 페이지에 함께 노출하여 사용자 편의성 향상

---

## 4. 운영자 관리 대시보드

### 4-1. 상태 박스 클릭 시 상세 페이지 이동
- 운영자 관리 대시보드의 다음 항목들의 **하위 세부사항 박스 클릭 시**, 해당 항목에 속한 물품 정보 목록 페이지로 이동 가능하도록 구현
  - `분실 신고 상태`
  - `습득물 상태`

---

## 5. 습득물 등록 화면

### 5-1. 일시 입력 UI 개선
- **`습득물 일시`** 입력 시 표시되는 **달력 + 시간 선택 창** 내부에 **`확인` 버튼** 추가

## 6. 매칭 / 수령 플로우 ⚠️ Breaking Change

### 6-1. 매칭 완료 후 상태 변경
- **현재**: 보관소 관리자 최종 승인 후에도 상태가 `매칭후보있음`으로 표시됨
- **변경**: 분실자가 매칭 승인 요청 → 보관소 관리자 최종 승인 시 → 상태를 **`찾기완료`** 로 변경
- 관련 상태 머신/DB 스키마/UI 모두 일관되게 반영

### 6-2. '내 신고' 메뉴에서 수령코드 확인 기능 추가
- **현재**: 보관소 관리자 승인 후 수령코드는 `알림` 메뉴에서만 확인 가능
- **추가**: `내 신고` 메뉴에서도 **수령코드를 볼 수 있는 버튼** 신규 생성
- 알림 메뉴와 내 신고 메뉴 양쪽에서 모두 확인 가능하도록 구현

### 6-3. 수령 완료 시 자동 종료 처리
- **현재**: 보관소 관리자가 3가지 정보 검증 → 수령 완료 확정 후, **종료 처리 대기 페이지**에서 수동으로 `종료 처리` 버튼을 눌러야 함
- **변경**: 3가지 정보 검증 + 수령 완료 확정 시 → **자동으로 물품 상태가 `종료`** 로 전환되도록 처리

---

## 7. 보관소 관리자 - 전체 습득물 관리 메뉴

### 7-1. '폐기예정 취소' 버튼 (토글 형태)
- 상태가 `폐기예정`인 물품의 경우, 기존 `폐기예정으로` 버튼이 있던 자리에 **`폐기예정 취소`** 버튼이 표시되도록 변경
- 즉, 물품 상태에 따라 같은 위치의 버튼이 토글되는 구조

### 7-2. 필터 교체: '장소' → '보관 위치'
- 현재 `장소` 필터가 있는 자리에 **`보관 위치`** 필터로 **교체** (위치는 동일)

---

## 8. 보관소 관리자 - 보관소 관리 메뉴

### 8-1. '빠른 진입' 섹션 삭제
- `보관소 관리` 메뉴 내 **`빠른 진입`** 섹션 제거
- 사유: 상단 박스 클릭 시 진입하는 동선과 **중복**

### 8-2. 폐기 기간 수동 설정 기능 추가
- **권한**: 운영자가 아닌 **보관소 관리자**가 설정
- **위치**: `보관소 관리` 메뉴 내 설정 버튼 신규 생성
- **기본값**:
  - 장기 미수령: **30일**
  - 수령 대기 자동 취소: **3일**
- 보관소 관리자가 위 값을 자유롭게 변경할 수 있도록 입력 UI 제공

## 💡 구현 시 참고 사항

- **상태 변경 항목**(매칭후보있음 → 찾기완료, 자동 종료 처리)은 **상태 머신 다이어그램을 먼저 정리**한 후 구현하는 것을 권장합니다.
- **권한 분리**(운영자 / 보관소 관리자 / 일반 사용자)는 미들웨어 또는 가드(Guard) 레벨에서 일관되게 처리되어야 합니다.
- **403 오류**는 초기 라우팅에서 역할별 홈 화면 매핑이 누락되었을 가능성이 큽니다 → 2-2 항목과 함께 처리하면 동시에 해결될 수 있습니다.
